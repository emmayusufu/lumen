# Security

If you find something that looks like a real security issue with Lumen, email me at kimaswaemma36@gmail.com instead of opening a public issue. I'd rather know quietly so I can fix it before someone hits it on a live instance.

## What to put in the report

* Where you saw it (URL or file path)
* The steps to reproduce, in order
* What actually goes wrong (what an attacker can do, however small)
* Your environment if it looks environmental

If you're not sure whether something counts as a security issue, send it anyway. Easier to thank you for a non-issue than miss a real one.

## What I'll do

I'll reply within a few days. The patch may take longer depending on the bug, but I'll keep you in the loop and credit you if you want credit. If the issue affects the live demo at lumen.jengahq.com I'll patch that first.

## In scope

* Anything in this repo
* The live demo at lumen.jengahq.com

## Out of scope

* Issues in upstream dependencies, file those with the project that owns the code
* Reports generated by automated scanners with no demonstration of impact
* Theoretical attacks that need physical access to the user's machine
* Social engineering or phishing aimed at the maintainer

## Disclosure

Once a fix has shipped, I'm happy to coordinate a public write-up if the bug is interesting. I won't sit on a patch waiting for a long embargo, this is a one-person project, not a CVE-grade vendor.
